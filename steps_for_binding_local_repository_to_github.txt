1. регистрируемся на гитхабе.
2. создаем там репозиторий какойнибудь
3. на локальном компьютере создаем ssh ключи:
- зачем то проверяем есть ли уже на компьютере ssh ключи. типа если есть, все удаляем
- ключи хранятся по умолчанию в папке .ssh, которая находится в домашнем каталоге пользователя.
В винде это C:\Users\Имя_пользователя
- Для генерации SSH-пары в терминале gitbash набираем команду:
$ ssh-keygen -t ed25519 -C "электронная почта, к которой привязан ваш аккаунт на GitHub"
или 
$ ssh-keygen -t rsa -b 4096 -C "электронная почта, к которой привязан ваш аккаунт на GitHub" 
пару раз жмем Enter на приглашения/подтверждения
затем надо будет придумать кодовую фразу, которую будем каждый раз вводить при push на гитхаб с локального компьютера
Теперь в папке .ssh появится 2 ключа: публичный и приватный: id_ed25519.pub и id_ed25519
Публичный - для зашифровки, его можно всем давать. Приватный - для расшифровки, он секретный, его никому давать нельзя.
Публичный имеет расширение .pub, приватный без расширения.

4. Дальше надо привязать SSH-ключ к GitHub:
- открываем файл id_ed25519.pub, там будет всего одна строчка вида:
ssh-ed25519 AAAAC3NzaC1lZDI1N555AAAAIOGbM2GY7R7CwxMI7l7CHMeSWKs1uj99zE1wqq+A8cro aavvnn@mail.ru, копируем ее в буфер обмена
- заходим на гитхаб, идем в настройки пользователя, ищем там раздел типа SSH and GPG keys.
в нем ищем кнопки/меню Добавить ssh ключ и вставляем из буфера скопированную строчку. Пишем название ключа в соотв. поле.

5. Для проверки что все получилось, на локальном компьютере в gitbash вводим:
$ ssh -T git@github.com 
вводим кодовую фразу
если все ОК, нам напишут чтото типа:
Hi aavvnn! You've successfully authenticated, but GitHub does not provide shell access.

6. Далее привязываем локальный репозиторий к репозиторию на гитхабе (называния локального и гитхабовского не обязаны совпадать при этом):
- заходим на гитхаб в нужный репозиторий и копируем ссылку ssh, она будет вида:
git@github.com:aavvnn/y_practical_work_1.git
- заходим в локальную консоль и вводим:
git remote add origin git@github.com:aavvnn/y_practical_work_1.git
- Убедиться, что репозитории связаны, — git remote -v, выдаст типа:
origin  git@github.com:aavvnn/y_practical_work_1.git (fetch)
origin  git@github.com:aavvnn/y_practical_work_1.git (push)


6. Далее можно просто вводить git push и наш локальный репозиторий будет забрасываться на гитхаб.
